# T-017.1: Housekeeping & Recovery Tool

**Status**: In Progress
**Priority**: 🚨 CRITICAL
**Owner**: Infra

## 📝 Description
Development of an analytical tool to identify and fix residual artifacts from failed volume operations (Shrink, Restore, etc.). This tool ensures cluster health by cleaning up stuck resources and "resurrecting" PVCs that are stuck in a terminating state due to failed operations.

## 🎯 Objectives
- [x] **Analytical Detection**: Identify 6 key issue types:
    1. Stuck Copy Jobs (`volume-copy-*`)
    2. Residual Temporary PVCs (`*-temp-*`)
    3. Unused "Restored" PVCs (`*-restored`)
    4. Stuck Terminating PVCs (Originals)
    5. Stuck Terminating PVCs (Temps)
    6. Unused/Detached PVCs (Scaled down workloads)
- [x] **Safety Mechanisms**:
    - **Snapshot-First**: Offer to snapshot before any deletion.
    - **PV Protection**: Enforce `Retain` policy on PVs before deleting stuck PVC objects.
- [x] **Resurrection Capability**: Logic to force-delete a stuck terminating PVC and immediately re-create it bound to the same PV, effectively "unsticking" it without data loss.
- [ ] **Code Reusability**: Refactor shared logic into `vm_utils.sh` to prevent duplication across Volume Manager scripts.

## 🛠️ Implementation Details
- **Script**: `scripts/volume_manager/housekeeping.sh`
- **Integration**: Accessed via Option 6 in the TUI Volume Manager (`k8s_ops_menu.sh`).
- **Dependencies**: `kubectl`, `jq`, `longhorn` (for snapshots).
